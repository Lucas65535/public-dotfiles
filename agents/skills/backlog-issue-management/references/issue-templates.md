# Issue Description Templates

Reusable **structural** templates for common issue types. These templates define the section layout — adapt the content language according to the Content Rules in SKILL.md (Summary in 日本語, Description body in 中文).

Use the appropriate template based on the issue type, and adapt content from user input.

## Bug Report

```
## Summary
[One-line description of the bug]

## Steps to Reproduce
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Expected Behavior
[What should happen]

## Actual Behavior
[What actually happens]

## Environment
- OS: [e.g., Windows 11, macOS 14]
- Browser: [e.g., Chrome 121]
- Version: [app version]

## Screenshots / Logs
[Attach or paste relevant evidence]

## Impact
[Severity assessment: who is affected and how]
```

## Task

```
## Objective
[What needs to be done and why]

## Acceptance Criteria
- [ ] [Criterion 1]
- [ ] [Criterion 2]
- [ ] [Criterion 3]

## Technical Notes
[Implementation hints, related code, API endpoints, etc.]

## Dependencies
[Other issues or external dependencies]
```

## Story (User Story)

```
## User Story
As a [role], I want to [action], so that [benefit].

## Acceptance Criteria
- [ ] Given [context], when [action], then [result]
- [ ] Given [context], when [action], then [result]

## UI/UX Notes
[Wireframes, mockups, design references]

## Technical Considerations
[Architecture impact, data model changes, API changes]

## Out of Scope
[Explicitly excluded items]
```

## Sub-Task

```
## Description
[Specific work item within the parent issue]

## Done Criteria
- [ ] [What marks this sub-task as complete]

## Notes
[Additional context or instructions]
```

## Improvement / Enhancement

```
## Current Behavior
[How it works now]

## Proposed Improvement
[What should change and why]

## Expected Benefit
[Performance gain, UX improvement, maintenance reduction, etc.]

## Implementation Approach
[High-level approach or options]
```
